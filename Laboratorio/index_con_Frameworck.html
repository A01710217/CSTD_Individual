Preguntas Lab 1

1.- ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la infraestructura física que permite la comunicación a través de la red mientras que la WWW es un conjunto de información y recursos accesibles a través de Internet.

2.- ¿Cuáles son las partes de un URL?

https://www.ejemplo.com/blog/articulo-1?param1=valor1¶m2=valor2#etiqueta1

  • Protocolo o Scheme (http, https)
  • Subdominio (WWW)
  • Dominio o Domain Name(ejemplo.com)
  • TLD (.com)
  • Ruta o Path(/blog/articulo-1)
  • Parámetros o Query Parameters (param1=valor1, param2=valor2)
  • Fragmento o Fragment (#etiqueta1)

3.- ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita datos de un recurso específico del servidor.
  • HEAD: Solicita los encabezados de un recurso específico del servidor.
  • POST: Envía datos para ser procesados a un recurso específico del servidor.
  • PUT: Reemplaza todos los datos de un recurso específico del servidor.
  • PATCH: Modifica parcialmente un recurso específico del servidor.
  • DELETE: Elimina un recurso específico del servidor.

4.- ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el método POST, ya que los datos enviados no se mostrarán en la barra de direcciones del navegador ni en el historial de navegación, lo que los hace más seguros para enviar información sensible como contraseñas.

5.- ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, el navegador solicita al servidor web que contiene la URL del recurso que se desea recuperar, y el servidor responde con el contenido del recurso solicitado, que luego se muestra en el navegador.

6.- Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 significa que la solicitud fue exitosa, no ocurrió ningún error.

7.- ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir el error 404, ya que significa que el recurso solicitado no fue encontrado en el servidor, lo que puede ser causado por un error en la URL o por un recurso que fue eliminado o movido.

8.- ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir el error 500, ya que significa que ocurrió un error interno en el servidor, lo que puede ser causado por un error en el código del servidor o por un problema con la configuración del servidor.

9.- ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que el atributo ya no se recomienda utilizar, ya que puede ser eliminado en futuras versiones de HTML. Algunos elementos de HTML 4 que en HTML5 están desaprobados algunos son: font, center, frame, frameset, noframes, big, strike, tt, acronym, applet, dir, isindex,basefont>, bgsound, menu, s, u.

10.- ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML 5

  • Tiene soporte para arrastrar y soltar

Sobre el ciclo de vida y desarrollo de los sistemas de información:

14.- ¿Cuál es el ciclo de vida de los sistemas de información?

Es el periodo de vida que tiene un sistema desde el momento de planeación hasta el momento de su descontinuación.

  1. Nace: cuando se comienza a detectar la oportunidad o nesesidad de crear u optimizar un sistema
  2. Desarolla o crece: cuando el sistema se diseña o se crea
  3. Produce: cuando se implementa, se prueba y se corrigen algunas fallas
  4. Muere: cuando el sistema ya no cumple las nesesidades, usa tecnología obsoleta o el sistema no se adapta a nuevas tecnologías

15.- ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información es un marco que describe las etapas por las que pasa un sistema de información desde su concepción hasta su eventual desmantelamiento.

  1. Planificación: Se determinan los requisitos del sistema.
  2. Análisis: Se realiza un estudio detallado de los requisitos del sistema y se diseña la arquitectura del sistema.
  3. Diseño del sistema: se diseñan las interfaces de usuario y se selecciona la base de datos.
  4. Implementación del sistema: se desarrolla el sistema de acuerdo con los diseños elaborados en la etapa de diseño.
  5. Pruebas del sistema: se llevan a cabo pruebas exhaustivas para garantizar que el sistema funcione correctamente y cumpla con los requisitos especificados.
  6. Mantenimiento del sistema: una vez que el sistema está en funcionamiento, se lleva a cabo su mantenimiento para corregir errores, mejorar el rendimiento y añadir nuevas funcionalidades según sea necesario.

Preguntas Lab 3

1.- Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

!important es usado para sobrescribir cualquier estilo que se le aplique a un elemento, sin importar la especificidad del selector. El uso de !important en CSS debe ser evitado siempre que sea posible, ya que puede causar problemas de mantenimiento y dificultar la comprensión del código.

  • El uso excesivo de !important puede hacer que el código sea difícil de mantener y entender.
  • El uso de !important puede causar problemas de especificidad en el código CSS, lo que puede llevar a conflictos y errores.
  • El uso de !important puede hacer que sea difícil de sobrescribir estilos en el futuro.

2.- Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

La elección imagen de fondo inapropiada puede afectar la legibilidad del texto y la usabilidad de la página.

  • Una imagen de fondo puede tener un gran impacto en la apariencia y la percepción de la página web.
  • Una imagen de fondo muy ocupada o con colores brillantes puede hacer que el texto sea difícil de leer.
  • Una imagen de fondo muy grande puede aumentar el tiempo de carga de la página.
  • Una imagen de fondo que no se repite correctamente puede causar problemas de diseño en la página.

3.- Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

  • Las unidades de porcentaje (%): son útiles para crear diseños que se adapten a un tamaño relativo.
  • Las unidades de píxeles (px): son útiles para crear diseños con tamaños fijos y precisos.
  • Las unidades de puntos (pt): son útiles para tamaños de texto y otros elementos que deben ser consistentes en diferentes dispositivos de visualización.

4.- ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Reduce el tamaño del archivo: La versión minimizada del CSS elimina los espacios en blanco, los comentarios y otros caracteres no esenciales del código CSS.

  • Reduce el tamaño del archivo.
  • Menor tiempo de transferencia.
  • Menor uso de ancho de banda y el consumo de recursos del servidor.

1.- ¿Qué diferencias y semejanzas hay entre Java y JavaScript?

  • Java es un lenguaje de programación de propósito general que se utiliza para desarrollar aplicaciones de escritorio, aplicaciones web, aplicaciones móviles, mientras que JavaScript es un lenguaje de programación de alto nivel que se utiliza principalmente para desarrollar aplicaciones web interactivas.
  • Java es un lenguaje de programación orientado a objetos, mientras que JavaScript es un lenguaje de programación basado en prototipos.
    • Java es un lenguaje de programación estáticamente tipado, mientras que es un lenguaje de programación dinámicamente tipado
  • Java se compila en bytecode que se ejecuta en la máquina virtual de Java (JVM), mientras que JavaScript se interpreta en el navegador web.
  • Java es un lenguaje de programación más antiguo que JavaScript, y tiene una sintaxis y características diferentes.

3.- ¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

  • push(): Añade uno o más elementos al final de un arreglo.
  • pop(): Elimina el último elemento de un arreglo.
  • unshift(): Añade al principio del arreglo
  • shift(): Elimina el primer elemento de un arreglo.
  • indexOf(): Devuelve el índice en el que se encuentra un elemento dado en el arreglo.

  • forEach(): Ejecuta una función dada una vez por cada elemento del arreglo.

4.- ¿Cómo se declara una variable con alcance local dentro de una función?

Una variable con alcance local dentro de una función se declara utilizando la palabra clave var, let o const, seguida del nombre de la variable y opcionalmente de un valor inicial.

Nota: Usar let o const en lugar de var es una buena práctica, ya que let y const tienen un alcance de bloque, mientras que var tiene un alcance de función.

5.- ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

  • Las variables globales pueden ser modificadas por cualquier parte del código, lo que puede llevar a comportamientos inesperados y errores difíciles de rastrear.
  • Las variables globales pueden ser sobrescritas por accidente, lo que puede causar problemas en otras partes del código.
  • Las variables globales pueden hacer que el código sea menos modular y reutilizable, ya que las funciones que dependen de variables globales pueden ser más difíciles de reutilizar en otros contextos.
